## What this module does

A bookkeeping firm that lives in the founder's head can never grow, never take a real
vacation, and never delegate without things slipping. The fix is documented process —
but writing SOPs is the work that always gets pushed to "someday."

The **SOP Builder Agent** turns any workflow you can describe (even messily, even out
loud) into three polished, ready-to-use documents.

```
   INPUT                          AGENT                       OUTPUTS
┌──────────────────────────┐                          ┌──────────────────────────┐
│ Any workflow, described  │                          │ 1. SOP (full procedure)  │
│ however you do it today  │  ──►  SOP Builder ─────►  │ 2. Checklist (do-it list)│
│ (rough is fine)          │       Agent (Claude/GPT)  │ 3. Training Document      │
└──────────────────────────┘                          └──────────────────────────┘
```

Describe how you onboard a client, run a close, or chase documents — talk it through in
bullet points or a brain-dump — and get back a clean SOP, a working checklist, and a
training doc you can hand to a new hire.

## Quick start (5 steps)

1. **Set up the Agent once.** Copy the prompt in `sop-builder-agent.md` into a Claude
   Project or ChatGPT custom GPT (instructions inside).
2. **Brain-dump the workflow.** Describe how you do it today — bullets, rough notes, or
   even a transcript of you talking it through. See `inputs/workflow-intake-guide.md`.
3. **Run the Agent.** It asks clarifying questions to fill gaps, then produces the SOP,
   checklist, and training doc.
4. **Review and adopt.** Confirm the steps match how you actually want it done, then save
   to your firm wiki / PM tool.
5. **Repeat** for each core workflow until your firm runs on documented process.

---

## ⚠️ A note on accuracy

The Agent documents *your* process — it doesn't invent bookkeeping best practice from
nowhere. Give it real steps; it organizes, clarifies, and fills obvious gaps (flagging
assumptions). **You confirm the procedure is correct before anyone follows it.** Don't
include client-identifying data in an SOP; keep them generic and reusable.
