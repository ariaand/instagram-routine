# Workflow Intake Guide — How to Describe a Workflow

**Module 5 input — Balance Operations Co. AI Bookkeeper Toolkit™**

**How to use:** You don't need a polished description to get a great SOP — the Agent
interviews you to fill gaps. But the more of this you provide up front, the fewer
questions it asks and the better the first draft. Brain-dump style is totally fine.

> 🔒 Keep SOPs generic. Describe the *process*, not a specific client's data — use roles
> and placeholders so the SOP is reusable across all clients.

---

## The fastest way: walk through the last time you did it

Don't describe the task in the abstract — narrate the most recent real instance, step by
step: "First I open QBO and go to… then I check… if it's like this I do X, otherwise Y…"
Concrete narration produces the best SOPs.

---

## What helps the Agent (give what you can)

### The basics
- [ ] **What's the workflow?** (e.g., "weekly transaction categorization for a client")
- [ ] **What triggers it / when does it start?** (a date, an email, a bank feed update)
- [ ] **What does "done" look like?** (the finished state)

### The steps
- [ ] The actual steps, in rough order (bullets are fine)
- [ ] **Decision points:** "if X, then Y" (e.g., "if a transaction is over $500 and I'm
      unsure, I flag it for the client")
- [ ] **Hand-offs:** does anyone else touch it? When?

### The details that make it usable
- [ ] **Tools & where things live** (QBO, bank feed, the portal, a spreadsheet, PM tool)
- [ ] **Quality checks** you do before calling it done
- [ ] **Common problems** and how you handle them
- [ ] **Deadline / cadence** (e.g., "every Friday," "by the 15th")
- [ ] **Rules:** anything that must always or never happen

---

## Copy-paste starter

```text
WORKFLOW: {{name + a rough description / brain-dump of the steps}}

TOOLS: {{QBO, bank feeds, payroll, portal, PM tool, etc.}}

ROLE & FREQUENCY: {{who does it — e.g., "staff bookkeeper" — and how often}}

RULES/STANDARDS: {{deadlines, must-always, never-do}}
```

## Example of a "good enough" brain-dump

You don't need to be tidy. This is plenty for the Agent to work with:

```text
WORKFLOW: Weekly categorization. Every Friday I go through each client's bank feed in
QBO. Most transactions auto-match from bank rules. The rest I categorize by hand. If I
don't know what something is I leave it in Ask My Accountant and add it to my client
question list. I check for duplicates and make sure transfers between accounts aren't
counted as income or expense. When the feed is clear I mark the client done in my tracker.

TOOLS: QBO, bank feeds/rules, my client tracker in ClickUp, the client portal for questions.

ROLE & FREQUENCY: Me (and soon a new staff bookkeeper). Weekly, every Friday.

RULES/STANDARDS: Never guess on anything over $500 — always ask. Feed must be cleared by
end of day Friday so nothing piles up for month-end.
```

The Agent will ask a few follow-ups (e.g., "how do you handle owner draws?" "what's your
bank-rule naming convention?") and then produce the SOP, checklist, and training doc.
